**Opt-in: squash planning commits on default branch before phase fork.** With `commit_docs: true`, every planning-doc change during `/gsd-discuss-phase` and `/gsd-plan-phase` lands as its own commit on the default branch. By the time `/gsd-execute-phase` forks the phase branch, the default branch carries a tail of N tiny `.planning/`-only commits. The new `git.squash_planning_commits_before_fork` flag (default `false`) makes `handle_branching` detect the contiguous `.planning/`-only run at HEAD and offer to consolidate it into a single `docs(planning): consolidate N planning commits` commit before the fork. Mixed commits (touching both `.planning/` and code) act as barriers — they are never rewritten. The squash is aborted with a warning if any candidate commit is already on `origin/<defaultBranch>`, and silently skipped when the run contains fewer than two commits or HEAD is not on the default branch. The user is shown the commit count and subjects and confirms yes/no — `TEXT_MODE` callers get a plain-text prompt.
